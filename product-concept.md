## 1. Product Vision (product-vision 스킬)

### 후보 비전 5개

1. "노인의 손목에서, 치매를 잠 못 들기 전에 발견합니다."
2. "가족이 멀리 있어도, 부모님의 뇌 건강을 매일 곁에서 지킵니다."
3. "기억이 사라지기 전에, 일상의 변화에서 신호를 듣습니다."
4. "Apple Watch 하나로, 치매 진료의 1년을 앞당깁니다."
5. "수면, 걸음, 대화 — 매일의 데이터로 부모님의 내일을 지킵니다."

### 선택: #2

> **"가족이 멀리 있어도, 부모님의 뇌 건강을 매일 곁에서 지킵니다."**

**선택 이유**:
- **Inspiring**: 핵심 정서적 동기(원거리 가족의 죄책감/걱정) 직접 자극
- **Achievable**: '진단'이 아닌 '지킴'으로 의료기기 규제 회피 + 24/7 모니터링은 워치로 충분히 가능
- **Emotional**: '곁에서'라는 단어가 물리적 거리감을 메우는 약속

## 2. Value Proposition (value-proposition 스킬, 6-part JTBD)

이 제품은 **이중 사용자(dual-sided)** 구조이므로 **두 개의 별도 value prop**이 필요합니다.

### 2-A. 노인 (실제 착용자)

| 항목 | 내용 |
|---|---|
| **Who** | 65세 이상, 경도인지장애(MCI) 위험군. 자녀와 떨어져 사는 경우 다수. 스마트폰 사용은 어렵지만 자식이 사준 워치는 받아들임. |
| **Why (JTBD)** | "내가 이상해지고 있는지 스스로는 모르겠고, 자식한테 짐이 되긴 싫다." 진행: **존엄을 잃지 않으면서 가족과 연결되어 있고 싶다**. |
| **What Before** | 1년에 한 번 보건소 인지선별검사(CIST). 그사이 가족이 "엄마 요즘 좀 이상해" 느낀 시점엔 이미 MCI 진행 중. 본인은 거부/부정. |
| **How** | Apple Watch만 착용. 별도 조작 X. 워치가 수면·걸음·심박 자동 수집. 주 1회 워치 화면에서 "오늘 본 사물 3개 말씀해주세요" 같은 30초 마이크로 인지테스트 (Solar LLM이 음성 평가). |
| **What After** | "내 손목이 알아서 건강 봐주니까 마음이 편하다." 의사 가는 횟수 ↓, 자녀와 통화에서 잔소리 ↓. 변화가 생겨도 본인이 인지하기 전에 가족이 알아서 챙김. |
| **Alternatives** | (1) 안 함 — 가장 흔함 (2) 가족이 영상통화로 살핌 — 비정기, 주관적 (3) 보건소 검사 — 1년 1회 (4) 노인 안심폰 — 위치만, 인지 X. |

### 2-B. 보호자 (실제 결제자)

| 항목 | 내용 |
|---|---|
| **Who** | 35–55세 자녀. 부모님과 다른 도시 거주. 직장/육아로 매일 못 봄. 부모님 치매에 대한 막연한 불안 + 의료 정보 격차. |
| **Why (JTBD)** | "엄마가 치매 초기인지 아닌지 객관적으로 알고 싶다. 너무 늦지 않게." 진행: **죄책감 없이, 객관적 근거로 부모를 챙기는 자녀가 되고 싶다**. |
| **What Before** | 카카오톡 안부 → "괜찮다"는 답만 받음. 명절에 만나면 "엄마 살 빠졌네, 말이 줄었네" 정도. 병원 모시고 가자 하면 거부당함. |
| **How** | iPhone 앱으로 부모님 주간 리포트 받음 (Solar LLM이 자연어로): *"이번 주 수면 효율이 78→65로 감소, 새벽 각성 횟수 3배 증가. 보행 속도도 8% 느려졌습니다. 가벼운 변화이지만 추적이 필요합니다."* 위험 신호 시 푸시 알림 + 병원 추천. |
| **What After** | 매일 데이터로 안심. 진짜 위험 신호 때만 행동. 부모와 영상통화에서 "지난주 잠 잘 못 주무셨네요" 같이 구체적인 대화 가능. **죄책감 → 능동적 케어**. |
| **Alternatives** | (1) 카톡 안부 — 무료지만 정보 0 (2) 보호자용 GPS 앱 — 위치만 (3) 요양보호사 방문 — 비싸고 본인 거부 (4) 치매안심센터 상담 — 상담 후 후속 X. |

### 통합 Positioning Statement (마케팅용)

> **"부모님은 워치만 차시면 됩니다. 변화가 생기면 우리가 먼저 알아챕니다.**
> 일상 속 수면·걸음·인지 데이터를 AI가 매일 분석해, 치매가 의심되는 순간 자녀의 휴대폰으로 알려드립니다."

## 3. Identify Assumptions (identify-assumptions-new 스킬, 8 risk categories)

각 가정에 대해 **신뢰도(1-10, 10이 가장 확신)** 와 **검증 방법**을 표기.

### 🔴 Value (가치)

| # | 가정 | 신뢰도 | 검증 방법 |
|---|---|---|---|
| V1 | 보호자는 부모의 일상 데이터 리포트에 월 ₩9,900~₩29,900을 지불한다 | 4 | 랜딩페이지 + 결제 인텐트 테스트(스모크 테스트) |
| V2 | 노인은 Apple Watch를 일상에서 24시간 착용하는 것을 받아들인다 | 5 | 5명 1주일 워치 착용 일지 (배터리 충전 시점, 답답함 등) |
| V3 | "조기 발견 = 가치"라는 인식이 한국 보호자에게 강하게 존재한다 | 7 | 치매안심센터 방문자 인터뷰 8명 |

### 🟡 Usability (사용성)

| # | 가정 | 신뢰도 | 검증 방법 |
|---|---|---|---|
| U1 | 65세+ 노인이 워치 화면의 인지테스트(예: 음성 답변, 도형 그리기) UI를 조작 가능하다 | 3 | 종이 프로토타입 → 5명 사용성 테스트 |
| U2 | 보호자가 "수면 효율 65%, HRV RMSSD 18ms" 같은 수치 없이 자연어 리포트만으로 충분히 이해한다 | 7 | 리포트 시안 5종 보호자에게 보여주고 행동의도 측정 |
| U3 | 알림 빈도가 주 1회 이내일 때 알림 피로(notification fatigue) 없이 행동을 유도한다 | 6 | 알림 빈도 A/B (주 1회 vs 격주) |

### 🔵 Viability (사업성)

| # | 가정 | 신뢰도 | 검증 방법 |
|---|---|---|---|
| B1 | Upstage Solar LLM API 비용이 사용자당 월 ₩3,000 이하로 유지된다 | 5 | 시나리오 시뮬레이션: 주간 리포트 1회 + 알림 분석 4회 = 토큰 추정 |
| B2 | 의료기기 인증 없이 "wellness 앱"으로 출시 가능하다 (식약처 가이드라인 회피) | 5 | 식약처 디지털헬스 SaMD 가이드라인 + 변호사 자문 30분 |
| B3 | Apple Watch + iPhone 동시 보유 가족이 한국에 충분히 많다 (최소 50만 가구) | 6 | KOSIS, 통신3사 보고서 데스크리서치 |
| B4 | 보호자가 결제하지만 노인이 거부 시 환불률이 30% 미만 | 3 | 베타 30가구 30일 추적 |

### 🟢 Feasibility (기술 실현성)

| # | 가정 | 신뢰도 | 검증 방법 |
|---|---|---|---|

| F2 | Solar LLM이 음성 답변에서 인지 저하 신호(언어 유창성, 단어 인출)를 의미있게 평가한다 | 3 | MMSE 점수가 라벨된 음성 데이터 50건으로 프롬프트 검증 |
| F3 | 첨부 샘플 데이터의 라벨(치매 고위험군) + 시계열 피처로 분류기를 만들면 AUC 0.75 이상 나온다 | 6 | 라벨링데이터 + 원천데이터로 baseline 모델(XGBoost) 학습 |

## 4. Prioritize Assumptions (prioritize-assumptions 스킬, Impact × Risk)

**Risk = (1 - 신뢰도/10) × Effort**, **Impact = 가정이 무너졌을 때 제품이 못 가는 정도**

### 매트릭스

```
                    낮은 위험 (확신 있음)        높은 위험 (확신 없음)
                    ─────────────────────       ─────────────────────
높은              │ ✅ 바로 구현                │ 🧪 즉시 실험 (최우선)
영향              │ V3, F4, G1                 │ V1, V2, F1, F2, F3,
                  │                            │  E1, E2, B2, U1, B4, T1
─────────────────┼────────────────────────────┼────────────────────────
낮은              │ 📌 나중에 (defer)          │ ❌ 폐기 검토
영향              │ U3, B3, S3, G3            │ S1, T2, B1, G2, S2,
                  │                            │  U2, E3
```

### 🧪 즉시 실험해야 할 가정 TOP 5 (High Impact × High Risk)

순서대로 검증 권장:

#### #1 — F3: 샘플 데이터로 baseline 모델 AUC 0.75 가능?
- **왜 먼저?** 이게 안 되면 제품 자체가 성립 안 함. 가장 큰 기술 리스크.
- **실험**: 첨부된 `라벨링데이터` + `원천데이터/2.수면`, `1.걸음거리`, `3.인지기능`으로 XGBoost baseline. 1주일.
- **성공 기준**: 5-fold CV AUC ≥ 0.75
- **실패 시**: 더 많은 피처/시계열 모델 필요 → 일정 +2개월

#### #2 — V2: 노인이 워치를 24/7 받아들이나?
- **왜?** 데이터 0이면 AI도 0. 모든 기술이 무용지물.
- **실험**: 부모님 또는 지인 노인 5명에게 1주일 Apple Watch 빌려주고 일지. 0원, 1주일.
- **성공 기준**: 일평균 18시간 이상 착용 ≥ 4명
- **실패 시**: 폼팩터 재고 (목걸이형? 반지형? Oura 협업?)

#### #3 — V1: 보호자 결제 의향 (₩9,900~₩29,900/월)
- **왜?** PMF의 핵심.
- **실험**: 랜딩페이지(framer/notion으로 1일) + 메타 광고 ₩100,000 → CTR/결제버튼 클릭률.
- **성공 기준**: 결제 인텐트 CTR ≥ 5%
- **실패 시**: 가격 재설계 또는 프리미엄 모델

#### #4 — F2: Solar LLM이 인지 평가에 쓸만한가?
- **왜?** Upstage API의 핵심 활용처. 안 되면 단순 ML 분석기로 격하.
- **실험**: MMSE 라벨 음성 50건 → Solar LLM에 평가 프롬프트 → 인간 평가자와 ICC 측정. 3일.
- **성공 기준**: ICC ≥ 0.7
- **실패 시**: Upstage는 리포트 생성용으로만. 인지 평가는 자체 ML.

#### #5 — E2: 거짓 양성 시 가족관계 손상 안 되는 메시지?
- **왜?** 이게 망가지면 한 번 잘못된 알림으로 입소문 망함.
- **실험**: 메시지 시안 5종을 보호자 10명에게 보여주고 "이거 받으면 어떤 행동?" 인터뷰. 1주.
- **성공 기준**: "당장 부모에게 화내며 전화" 같은 부정 반응 < 20%
- **실패 시**: 알림 임계값 보수적으로 + UX 재설계

### ✅ 바로 구현 가능한 가정 (High Impact × Low Risk)

- **F4** — HealthKit Background Delivery: Apple 공식 문서대로 구현
- **G1** — 보호자 마케팅 채널: 인스타/카페 광고 진행
- **V3** — 한국 보호자의 조기발견 가치 인식: 이미 강함 (치매국가책임제 영향)

### ❌ 재검토 필요한 가정 (Low Impact × High Risk)

- **S1, T2, B1**: 임팩트 대비 리스크 높음 → 베타 이후 다시 평가
- **U2 (수치 없는 자연어 리포트)**: 보호자 중 일부는 수치를 원할 수 있음. 두 가지 모두 제공하는 것으로 사이드스텝.
